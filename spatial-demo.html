How it works. Each spot is a check-in node in a small verified map (nodes + edges). Surroundings and directions are produced by shortest-path routing over that graph, so the guide narrates the real route and says so plainly when something isn't mapped — the wayfinding equivalent of an object that declines to guess.
Nothing here calls a server. No camera, no tracking, no cloud maps API — the whole thing runs on this device. Sovereign by default. Connected by choice.
