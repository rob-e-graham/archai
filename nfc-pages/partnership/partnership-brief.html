Overview

What is ARCHAI?

ARCHAI is an infrastructure platform that connects open-access museum collections to a conversational AI layer, then deploys that layer as scannable NFC and QR pages — on physical objects, printed posters, gallery labels, or anywhere with a surface and a wall.

Every object in the ARCHAI system generates a living visitor page. Scan a code, ask a question, get a real answer sourced from the collection record. The conversation runs on a local language model — no visitor data leaves the device, no third-party API in the loop.

"We're not building a competing collection site. We're building the next layer — conversational, physical, deployable, open."

Architecture

How it works

The ARCHAI stack is three layers:

  1. Harvest layer — harvesters pull open-access data from museum APIs (CC0, CC BY, OGL). Every record passes a legal gate before reaching the collection. Embeddings generated via Ollama (nomic-embed-text) stored in Qdrant.
  2. Conversation layer — object queries use vector search (cosine similarity) to retrieve context, then pass it to a local LLM (qwen2.5:32b or compatible). No cloud API required. No visitor data transmitted.
  3. Deployment layer — each object generates an HTML visitor page (AUX.IO). Pages are static-first, mobile-first, and scannable via NFC tag or QR code. Downloadable posters in 5 sizes.
Ollama
Local LLM inference. qwen2.5:32b for conversation. nomic-embed-text for embeddings.
Qdrant
Vector database. Cosine similarity search across 20K+ object embeddings.
Node.js / Express
Backend API. Proxy layer between visitor pages and local Ollama.
IIIF
Image serving. Rijksmuseum, AIC, Wellcome, Getty, Tate — all via standard IIIF endpoints.
NFC / QR
Physical deployment. NFC-212 tags or printed QR codes trigger visitor pages on any device.
Canvas API
In-browser poster generation. A4–A0–sticker–postcard PNG at 150–300 dpi.

Print Deployment

Downloadable poster system

Every object page includes a download button for high-resolution printable posters. Generated entirely in-browser via Canvas API — no server, no upload, no account. Output is a PNG with the object image, title, artist, institution credit, QR code, and ARCHAI branding.

Legal & Ethics

Open access, properly done

ARCHAI takes rights seriously. Every source in our pipeline passes a legal harvest gate before data is indexed. The rules are explicit and enforced in code:

  • CC0 or equivalent open licence required for any image displayed publicly on AUX.IO
  • CC BY attribution stored in every Qdrant payload and rendered on every visitor page
  • Metadata-only mode for sources where images are not cleared (Whitney, MoMA, Harvard)
  • Cultural sensitivity reviews flagged in the collection matrix for Pacific, Indigenous, and sacred material
  • No API keys committed — all credentials via macOS Keychain or environment variables
  • Local LLM only — no visitor conversation data sent to third-party servers

The full collection targets matrix, legal harvest bot, and rights verification status are documented and available on request.
