## Pre-send gate (complete before the first email)

1. Merge the open documentation/correctness PR so the public repo shows the
   current licence, provenance, and voice-layer wording.
2. Attach the correct public white paper version to every draft.
3. Check each draft describes the project as *built on open-source components
   with an open-source GLAM toolkit release as the goal* — not as
   currently open-source licensed (LICENSE is all rights reserved during the
   research period).
4. Voice claims in any draft: browser speech today, self-hosted Whisper/Piper
   in development.
5. Every email: preserve the correction/takedown offer paragraph — it is the
   strongest trust signal for source institutions.

## Discipline

- One email at a time, individually addressed; no bulk send.
- Log send date + follow-up date (10–14 days) in `OUTREACH_TRACKER.md`.
- Any rights or correction request outranks partnership discussion — action
  it first and log it in the Correction / Takedown Log.
- Keep all wording factual and courteous; assume every message may be read
  by the institution's legal team.
