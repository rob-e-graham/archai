## ToMuCo (Tokyo Museum Collection) — HELD ON LEGAL GROUNDS

`scripts/tomuco-harvester.js`. **Not onboarded — there are zero legally-usable
image-backed records.**

Investigation:
- ToMuCo terms (museumcollection.tokyo/en/terms/): default CC BY / Public Data
  License, some CC0/Public Domain Mark — BUT many images carry third-party rights
  where **the user must obtain consent unless rights are explicitly cleared**.
- Sampled the 6 museums in the harvester: tobikan, topmuseum, tatemonoen,
  teien-art-museum, edo-tokyo-museum populate **no** `schema:license` at all.
  Only mot-art-museum has explicit licenses (13 Public Domain Mark items in 150).
- **Critical:** of mot-art-museum's 13 open-license items, **0 have images.** The
  image-backed items have no license; the licensed items have no image. The
  legally-safe intersection is empty.

Action taken: added a hard **legal gate** (`hasOpenLicense()`) to the harvester —
it now skips any item without an explicit CC0 / Public Domain Mark / CC BY license.
With the gate, the harvester correctly yields 0 records. The gate stays in place so
ToMuCo can never be onboarded without clear licensing. If ToMuCo later exposes
image-backed open-license items, the harvester will pick them up automatically.

**Decision for Rob:** ToMuCo is off the table until the museum publishes
image-backed openly-licensed records. This is the correct call for a
sovereignty/ethics-framed project — do not override the gate.
